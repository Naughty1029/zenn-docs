1. group とは何か

通常の CSS（および Tailwind）は「ある要素にホバーしたらその要素自身のスタイルを変える」のが基本です。 でも実際の UI では「カードにホバーしたら、その中のタイトルだけ色を変えたい」というケースがよくあります。

純粋な CSS で書くと .card:hover .card-title { color: blue; } のように親セレクタを使いますが、 Tailwind はユーティリティクラスだけで CSS を書くのが原則のため、これを直接書く場所がありません。

そこで登場するのが group です。
親要素に group クラスをつけ、子要素で group-hover: などのプレフィックスを使うことで、 「親の状態に応じて子のスタイルを変える」が Tailwind だけで完結できます。

ポイント
group は「親の状態を子に伝えるための仕組み」です。

2. 基本的な使い方（group-hover）

もっとも使われるのは group-hover: です。実際に手を動かしながら覚えましょう。

手順

  1. スタイルを変化のきっかけにしたい親要素に group クラスをつける
  2. スタイルを変えたい子要素に group-hover:Tailwindクラス をつける
  3. デフォルト状態のクラスも忘れずに書く（例：opacity-0 → group-hover:opacity-100）
<!-- 親に group をつける -->
<div class="group cursor-pointer border rounded-xl p-4">

  <!-- ホバーで文字色を青に変える -->
  <h2 class="text-gray-800 group-hover:text-blue-600">
    カードタイトル
  </h2>

  <!-- ホバーで「続きを読む」を表示 -->
  <p class="opacity-0 group-hover:opacity-100 text-sm text-blue-500">
    続きを読む →
  </p>

</div>
ライブデモ — カードにホバーしてみて
カードタイトル
続きを読む →
書き方のコツ
group-hover: の後には通常の Tailwind クラスをそのまま書けます。
例：group-hover:scale-105、group-hover:bg-blue-50、group-hover:translate-x-1

group-hover 以外のバリアントも使える

hover 以外にも、group-focus:、group-active:、group-disabled: なども同じ要領で使えます。

<!-- インプットにフォーカスが当たったらラベルを強調 -->
<div class="group">
  <label class="text-gray-500 group-focus-within:text-blue-600">
    メールアドレス
  </label>
  <input type="email" class="border rounded p-2 w-full" />
</div>

3. 兄弟要素への伝達：peer との違い

group と混同しやすいのが peer です。 違いを一言で言うと：

機能関係典型的なユースケース
group親 → 子カードホバーで中のテキストを変える
peer兄弟 → 次の兄弟チェックボックスの状態でラベルを変える
 
<!-- peer: 兄弟要素の状態を次の兄弟に伝える -->
<label>
  <!-- 状態の発信元に peer をつける -->
  <input type="checkbox" class="peer" />

  <!-- peer-checked: でチェック時のスタイルを指定 -->
  <span class="peer-checked:line-through peer-checked:text-gray-400">
    牛乳を買う
  </span>
</label>
ライブデモ — チェックしてみて
peerの制約
peer は 後ろの兄弟要素 にしか影響できません。HTML の順番が重要です。 チェックボックスをラベルの後に書くと効かないので注意してください。

4. 名前付き group（group/名前）

コンポーネントがネストされると、「どの group 親を参照しているか」が曖昧になります。 たとえばカードの中にボタンがあって、「カードにホバー」と「ボタンにホバー」で別々の動きをさせたいケースです。

そこで グループに名前をつける ことができます。 名前は group/card のようにスラッシュ区切りで自由につけられます。 

<!-- 外側のグループに "card" という名前をつける -->
<div class="group/card">

  <!-- /card の親にホバーで反応 -->
  <h2 class="group-hover/card:text-blue-600">
    カードにホバーで青くなる
  </h2>

  <!-- 内側のグループに "btn" という名前をつける -->
  <button class="group/btn">

    <!-- /btn の親にホバーで反応 -->
    <span class="group-hover/btn:text-red-500">
      ボタンにホバーで赤くなる
    </span>

  </button>
</div>
ライブデモ — カード全体 vs ボタン、ホバーの反応が別々
カードにホバー → このテキストが青くなる
ボタンにホバー → これが赤くなる

名前がない場合は一番近い祖先の group を参照するため、入れ子が深くなると意図しない動作になることがあります。ネストするときは名前をつける習慣をつけておくと安全です。

5. data 属性との組み合わせ

group は hover などの疑似クラスだけでなく、HTML の data 属性 とも組み合わせられます。 これを使うと「同じ HTML 構造でも data 属性の値によって見た目を切り替える」ことが CSS も JS も書かずに実現できます。

最初に見たチャット UI のコードはまさにこの活用例です。 

<!-- data-role の値に応じてスタイルを変える -->
<div class="group/message" data-role="user">

  <div class="
    group-data-[role=user]/message:bg-zinc-200
    group-data-[role=user]/message:rounded-xl
    group-data-[role=user]/message:px-3
    group-data-[role=user]/message:py-2
    group-data-[role=user]/message:ml-auto
  ">
    メッセージ内容
  </div>

</div>

<!-- data-role="assistant" の場合は上のクラスがすべて無効になる
     → 背景なし・左寄せのシンプルな表示になる -->

Rails などのテンプレートエンジンを使えば、data-role を動的に埋め込むだけでスタイルが自動的に切り替わります。

<%# Rails ERB の例 -->
<div class="group/message" data-role="<%= message[:role] %>">
  ...
</div>

6. まとめ：どれをいつ使うか

機能使うときキーワード
group / group-hover: 親にホバーして子のスタイルを変えたい カードホバー、リストアイテム
group/名前 group がネストしていてどの親か区別したい コンポーネント内の複数 group
peer / peer-checked: 兄弟要素の状態を次の兄弟に伝えたい チェックボックス、ラジオボタン
group-data-[attr=val]: data 属性の値でスタイルを動的に切り替えたい チャット UI、ロール別スタイル

最初のコードを読み返してみよう

記事冒頭のチャット UI のコードに戻ると、group/message・group-data-[role=user]/message: がすべて読めるはずです。

group/message で名前付きグループを定義し、data-role の値が user のときだけ 背景色・角丸・パディングを適用する。 Rails 側は <%= message[:role] %> を差し込むだけ。これだけで JS も CSS も追加せずに user / assistant の見た目を切り替えています。
