# Claude記事作成・編集ガイドライン

Claudeが技術記事を作成・編集する際の汎用的なルールとベストプラクティス

## 記事構成の原則

### 1. 読者に優しい段階的構成
- **概要 → 詳細 → 具体例** の順で構成する
- いきなり詳細な技術的説明や具体例から始めない
- 読者が「なぜ必要なのか」を理解してから「どうやるのか」に進む

### 2. 個人的な体験・動機の活用
- 「はじめに」で筆者の体験や疑問を共有する
- 「〜という問題に遭遇したため」「〜という疑問を持っていました」
- 読者との共感を生み、記事を読む動機を提供する

### 3. セクション間のつなぎを重視
- 各セクションの終わりに次のセクションへの橋渡しを入れる
- 「では、〜について詳しく見ていきましょう」
- 「それでは、これまでの説明を実際のコードで確認してみましょう」
- 「ここまで〜について説明してきましたが、次は〜を理解する必要があります」

### 4. 重複の排除
- 同じ内容や概念の説明を複数箇所で繰り返さない
- 必要な場合は「先ほど説明した〜」として参照する
- 複数のセクションで類似の説明がある場合は統合を検討する

## 文体とトーン

### 1. 自然な日本語
- **コロンを多用しない**（AIが書いたような印象を与えるため）
- 「〜について：」「〜の意味：」などは避ける
- 「〜について」「〜の意味は次の通りです」など自然な表現を使う

### 2. 読者との距離感
- 専門用語は必要に応じて説明を加える
- 読者の立場に立った説明を心がける
- 「私たち」「あなた」など適度な親近感を持たせる

### 3. 具体例の扱い
- 具体例には必ず**注釈や補足説明**を加える
- 「実際には〜」「ここでは〜のみを表示していますが」
- 読者が誤解しないよう、簡略化している部分を明記する

## 技術記事特有の注意点

### 1. コマンド例の説明
- コマンド例を出す際は、実際の動作環境では他にも多くの処理が発生することを注釈で明記
- 簡略化している理由を説明する

### 2. 理論と実践のバランス
- 理論的説明の後に必ず実践例を提供
- 抽象的な概念は具体例で補強
- 全体像を示してから詳細に入る

### 3. 効果的な説明手法
- **ビフォーアフター比較** 新技術の効果を具体的に示す
- **権威ある引用** 専門的な内容に説得力を付与
- **messageボックス活用** 重要な技術的詳細や注意事項を整理

### 4. セクション構成の一例
```
1. はじめに（問題提起・個人体験・目的）
2. 基本概念（何か・なぜ必要か）
3. 詳細説明（どのように動作するか）
4. 全体像（システム全体での位置づけ）
5. 具体例（実際の動作確認・ビフォーアフター）
6. まとめ（重要ポイントの再確認）
```

## 避けるべき表現・構成

### 1. AIらしい表現
- 過度なコロンの使用
- 「以下の通りです：」
- 機械的な箇条書きの連続

### 2. 読者を混乱させる構成
- いきなり詳細な技術説明
- 概念説明なしの具体例
- 専門用語の説明不足

### 3. 情報の羅列
- つなぎの言葉がない連続したセクション
- 前後の文脈を無視した説明
- 重複した内容の繰り返し

## チェックリスト

記事完成時の確認事項：

- [ ] セクション間につなぎの言葉があるか
- [ ] 重複した説明がないか
- [ ] 不要なコロンを使っていないか
- [ ] 具体例に適切な注釈があるか
- [ ] 概要→詳細→具体例の流れになっているか
- [ ] 読者の理解レベルに合った説明か
- [ ] 専門用語に説明があるか
- [ ] 個人的な体験・動機が含まれているか
- [ ] messageボックスを効果的に活用しているか
- [ ] ビフォーアフター比較で効果を示しているか

## 修正時の対応

### フィードバックを受けた際の対応手順
1. 現在の構成を分析
2. 問題点を特定
3. 改善案を提示
4. ユーザーの承認後に修正実行
5. つなぎの言葉や注釈の追加

### よくある修正パターン
- 構成の並び替え（概要→詳細→具体例の順に）
- 重複説明の統合
- セクション間のつなぎ追加
- コロンの除去
- 注釈の追加

## Zenn記事特有の要素

### messageボックスの活用
```markdown
:::message
重要な技術的詳細や注意事項を整理
:::
```

### 権威ある引用の追加
専門的な内容には、信頼できる出典からの引用を含める

### ビフォーアフター比較
技術の効果を具体的に示すため、従来の方法と新しい方法を比較
