## はじめに

Tailwind CSS でカードコンポーネントを作っていたとき、「カード全体にホバーしたら、中のタイトルだけ色を変えたい」という場面がありました。通常の `hover:` だと、その要素自身にマウスを乗せたときにしかスタイルが変わりません。カード全体のホバーに反応させるにはどうすればいいのか、調べてみると `group` という仕組みにたどり着きました。

この記事では、Tailwind CSS の `group` 機能について、基本的な使い方からネストした場合の名前付き group、さらに data 属性との組み合わせまでを段階的に解説していきます。

## group とは何か

通常の CSS や Tailwind では、「ある要素にホバーしたら**その要素自身**のスタイルを変える」のが基本です。

```html
<!-- hover: はその要素自身にしか効かない -->
<div class="hover:bg-blue-50">
  <h2>カードタイトル</h2>  <!-- ← divにホバーしてもh2は変わらない -->
</div>
```

でも実際の UI では、「カードにホバーしたら、中のタイトルだけ色を変えたい」「リストアイテムにホバーしたら、右端のアイコンを表示したい」といったケースがよくあります。

純粋な CSS で書くと `.card:hover .card-title { color: blue; }` のように親セレクタを使いますが、Tailwind はユーティリティクラスだけで CSS を書くのが原則なので、これを直接書く場所がありません。

そこで登場するのが **group** です。

**親要素に `group` クラスをつけ、子要素で `group-hover:` などのプレフィックスを使う**ことで、「親の状態に応じて子のスタイルを変える」が Tailwind だけで実現できます。

:::message
group は「親の状態を子に伝えるための仕組み」です。
:::

## 基本的な使い方（group-hover）

もっとも使われるのは `group-hover:` です。手順はシンプルで、3ステップで使えます。

1. スタイル変化のきっかけにしたい**親要素**に `group` クラスをつける
2. スタイルを変えたい**子要素**に `group-hover:○○` をつける
3. デフォルト状態のクラスも忘れずに書く（例：`opacity-0` → `group-hover:opacity-100`）

```html
<!-- 親に group をつける -->
<div class="group cursor-pointer border rounded-xl p-4">

  <!-- ホバーで文字色を青に変える -->
  <h2 class="text-gray-800 group-hover:text-blue-600">
    カードタイトル
  </h2>

  <!-- ホバーで「続きを読む」を表示 -->
  <p class="opacity-0 group-hover:opacity-100 text-sm text-blue-500">
    続きを読む →
  </p>

</div>
```

この例では、`div` 全体にマウスを乗せると、中の `h2` が青くなり、`p` がフェードインして表示されます。`div` の外にマウスが出ると元に戻ります。

`group-hover:` の後には通常の Tailwind クラスをそのまま書けるので、`group-hover:scale-105` で少し拡大させたり、`group-hover:bg-blue-50` で背景色を変えたり、自由に組み合わせられます。

### group-hover 以外のバリアントも使える

hover だけでなく、他の状態にも対応しています。

```html
<!-- インプットにフォーカスが当たったらラベルを強調 -->
<div class="group">
  <label class="text-gray-500 group-focus-within:text-blue-600">
    メールアドレス
  </label>
  <input type="email" class="border rounded p-2 w-full" />
</div>
```

`group-focus:` や `group-active:`、`group-disabled:` なども同じ要領で使えます。フォームのラベルをフォーカスに連動させたり、ボタンが無効状態のときに中のアイコンをグレーアウトさせたり、いろいろな場面で役立ちます。

それでは、`group` と似ている `peer` との違いを見ていきましょう。

## 兄弟要素への伝達 — peer との違い

`group` と混同しやすいのが **peer** です。どちらも「ある要素の状態を別の要素に伝える」仕組みですが、伝わる方向が違います。

| 機能 | 関係 | 典型的なユースケース |
|------|------|---------------------|
| `group` | 親 → 子 | カードホバーで中のテキストを変える |
| `peer` | 兄弟 → 次の兄弟 | チェックボックスの状態でラベルを変える |

`peer` の使い方も `group` とよく似ています。状態の発信元に `peer` をつけ、影響を受ける要素に `peer-checked:` などのプレフィックスを書きます。

```html
<!-- peer: 兄弟要素の状態を次の兄弟に伝える -->
<div>
  <!-- 状態の発信元に peer をつける -->
  <input type="checkbox" id="task1" class="peer" />

  <!-- peer-checked: でチェック時のスタイルを指定 -->
  <label for="task1" class="peer-checked:line-through peer-checked:text-gray-400">
    牛乳を買う
  </label>
</div>
```

チェックボックスにチェックを入れると、隣の `label` に取り消し線がつき、テキストがグレーになります。TODO リストのような UI を、JavaScript なしで実現できるわけです。

:::message alert
peer は**後ろの兄弟要素**にしか影響できません。HTML の記述順が重要です。チェックボックスをラベルの後に書くと効かないので注意してください。これは CSS の `~`（後続兄弟セレクタ）を使っているためです。
:::

ここまでで `group` と `peer` の基本的な違いを押さえました。次は、`group` がネストしたときに起きる問題と、その解決策を見ていきます。

## 名前付き group（group/名前）

コンポーネントがネストされると、「どの `group` を参照しているか」が曖昧になる場面が出てきます。

たとえば、カードの中にボタンがあって、「カードにホバーしたときの動き」と「ボタンにホバーしたときの動き」を別々に制御したい場合を考えてみましょう。

```html
<!-- group が入れ子になると、子要素がどの group を参照するか曖昧になる -->
<div class="group">
  <h2 class="group-hover:text-blue-600">カードタイトル</h2>

  <button class="group">
    <span class="group-hover:text-red-500">削除</span>
    <!-- ↑ カードの group？ ボタンの group？ -->
  </button>
</div>
```

この問題を解決するのが**名前付き group** です。`group/card` のようにスラッシュ区切りで名前をつけられます。

```html
<!-- 外側のグループに "card" という名前をつける -->
<div class="group/card">

  <!-- /card のホバーで反応 -->
  <h2 class="group-hover/card:text-blue-600">
    カードにホバーで青くなる
  </h2>

  <!-- 内側のグループに "btn" という名前をつける -->
  <button class="group/btn">

    <!-- /btn のホバーで反応 -->
    <span class="group-hover/btn:text-red-500">
      ボタンにホバーで赤くなる
    </span>

  </button>
</div>
```

`group-hover/card:` は「`group/card` の親がホバーされたとき」、`group-hover/btn:` は「`group/btn` の親がホバーされたとき」という意味になります。これでどの group を参照しているかが明確になります。

名前がない場合は一番近い祖先の `group` を参照するため、シンプルな構成なら名前なしでも問題ありません。ただ、入れ子が深くなると意図しない動作になることがあるので、ネストするときは名前をつけておくと安全です。

では最後に、group のもう一つの活用法として data 属性との組み合わせを見ていきましょう。

## data 属性との組み合わせ

`group` は hover などの疑似クラスだけでなく、**HTML の data 属性**とも組み合わせられます。これを使うと「同じ HTML テンプレートでも、data 属性の値によって見た目を切り替える」ということが実現できます。

たとえば、チャット UI を考えてみます。ユーザーのメッセージは右寄せで背景色あり、アシスタントのメッセージは左寄せで背景色なし、という切り替えを `data-role` 属性で制御できます。

```html
<!-- data-role の値に応じてスタイルを変える -->
<div class="group/message" data-role="user">

  <div class="
    group-data-[role=user]/message:bg-zinc-200
    group-data-[role=user]/message:rounded-xl
    group-data-[role=user]/message:px-3
    group-data-[role=user]/message:py-2
    group-data-[role=user]/message:ml-auto
  ">
    メッセージ内容
  </div>

</div>

<!-- data-role="assistant" の場合は上のクラスがすべて無効になる
     → 背景なし・左寄せのシンプルな表示になる -->
```

`group-data-[role=user]/message:bg-zinc-200` の読み方は次のとおりです。

- `group-data-[role=user]` → data-role が "user" のとき
- `/message` → "message" という名前の group を参照
- `:bg-zinc-200` → 適用するスタイル

Rails の ERB や React の JSX などテンプレートエンジンと組み合わせれば、`data-role` を動的に埋め込むだけでスタイルが自動的に切り替わります。

```erb
<%# Rails ERB の例 %>
<div class="group/message" data-role="<%= message[:role] %>">
  ...
</div>
```

:::message
data 属性との組み合わせは、条件分岐のロジックを HTML/CSS 側に寄せられるのがメリットです。JavaScript で className を分岐させたり、テンプレート内で if 文を書いてクラスを出し分けたりする必要がなくなります。
:::

## まとめ

`group` の各機能を整理すると、次のように使い分けられます。

| 機能 | 使うとき | 典型的なユースケース |
|------|---------|---------------------|
| `group` / `group-hover:` | 親にホバーして子のスタイルを変えたい | カードホバー、リストアイテム |
| `group/名前` | group がネストしていてどの親か区別したい | コンポーネント内の複数 group |
| `peer` / `peer-checked:` | 兄弟要素の状態を次の兄弟に伝えたい | チェックボックス、ラジオボタン |
| `group-data-[attr=val]:` | data 属性の値でスタイルを動的に切り替えたい | チャット UI、ロール別スタイル |

最初は `group` と `group-hover:` だけ覚えておけば、ほとんどのケースに対応できます。ネストが出てきたら名前付き group、動的なスタイル切り替えが必要になったら data 属性との組み合わせ、という順番でステップアップしていけば無理なく使いこなせるようになるはずです。
